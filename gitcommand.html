1. git config: Used to set the git configuration values.

2. git init: To create a new git repository.

3. git clone: Enable the cloning of the git repository.

4. git add: Helps to add the file contents to staging.

5. git mv: Used to rename or move the repository files.

6. git branch: Used to create, list or delete branches.

7. git diff: Helps compare the changes in the files.

8. git rm-f: Used to remove files from git repository.

9. git status: Provides information on the staging area and repository.

10. git log: Used to filter git logs and show commit history.

11. git checkout: Helps to switch between branches.

12. git merge: Helps to change/merge the git branches.

13. git fetch: Used to download remote git repository.

14. git push: Helps update remote branch along with local commits.

15. git revert: Used to change/revert the present commits.

16. git pull: Helps to download and merge the git repository.

17. git rebase: Used to change/modify various commits.

18. git stash: Helps to locally save the uncommitted changes.

19. git reset: Used to revert the git repository commits.

20. git reflog: Helps monitor the branch changes.

21. git archive: Helps to create an archive file from various git references.

22. git remote add: Used to view, create and delete connections, mostly to other repositories.

23. git remote remove: Helps remove the remote URL from the repository.

24. git blame: Used to examine a file’s contents in detail and view the author behind the modifications.

25. git describe: Helps to find a recent tag that can be reached from a commit.

26. git rm: Also known as git remove, it is used to remove files from a git repository.

27. git bisect: Helps to check out a commit that is between different states.

28. git fetch – all: Helps fetch the remotes and branches that are registered.

29. git grep: Helps to search a specific content on the git repository.

30. git config – local: Used to arrange the git configuration values, be it for a project or global level.

31. git branch – track: Used to create a relationship to the remote branch:

32. git push – delete: Helps to delete a remote branch.

33. git fetch – prune: Helps to get rid of branches that are outdated.

34. git workout: Used to work in multiple Git branches as well as checkout at the same time.

35. git merge – squash: Used to minimize the git history of branches upon request.

36. git push – force: Helps to overwrite the remote branch.

37. git add – all: Helps to maintain the code version.

39. git apply: Helps to apply the output of a git command in the form of a new commit in a direct manner.

40. git shortlog: Shows a summarized view of the git log output.

41. git clea: Used to delete the files that are untracked in the local working tree.

42. git show-ref: Shows the references and related commit IDs that are available in a local repository.

43. git filter-branch: Helps to modify a repository’s history.

44. git gc: Gc stands for garbage collection and the command is used for repository maintenance.

45. git check-ignore: This inbuilt command helps to see if a file is ignored or not.

46. git fast-import: Used to exchange data between control systems that are of different versions.

47. git instaweb: Helps to browse git repository in a web browser.

48. git svn: Supports the interaction of git with Subversion repositories.

49. git cat-file: Used to obtain command tag contents which are displayed to users.

50. git fsck: Helps to find and restore commits and files that are lost.
